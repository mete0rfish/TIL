# 개요
대용량 데이터를 분산 처리하기 위해 나누어서 다시 합치는 패턴이다.
쉽게 말해 많은 데이터를 여러 대의 컴퓨터에 나눠주고 계산하게 한 뒤, 그 결과를 모은다.
이 패턴은 Hadoop과 같은 빅데이터 처리 프레임워크 및 현대 함수형 프로그래밍의 근간이 된다.

# 아키텍처
1. Master가 입력 키-값 쌍의 전체 세트를 M개의 청크로 나눠줌.
2. 전체 데이터 세트를 M개의 다른 머신을 나누어 병렬 실행
3. 맵 함수 실행 결과를 R개의 리전 별로 intermediate storage에 저장
4. Master가 다시 R개의 청크를 R개의 워커에게 전달
5. R개의 Reduce 워커가 병렬 실행
6. 최종 결과를 전역 엑세스 가능한 형태로 출력

# 장점
- 같은 아키텍처 재사용 가능
- 처리 Task를 확장해 원하는 만큼 스케일 아웃이 가능
- 워커가 병렬로 작동함

즉, 많은 양의 데이터를 짧은 시간안에 가공이 가능하다.

# 실패와 복구
- 많은 워커가 돌아감 -> 데이터 가공 중 실패 확률이 높음
  - 이를 해결하기 위해 Master가 Map 또는 Reduce 워커의 상태를 주기적으로 체크
- 그런데 Master가 고장나는 경우도 있음
  - 이 경우 다른 새로운 마스터를 실행 시킴 -> 미리 스냅샷을 통해 빠르게 스위칭

# 클라우드 환경에서
### 사용 경우
1. 필요한 만큼의 머신에 바로 엑세스 가능한 경우
2. 일괄 처리 패턴의 일종
    - 즉, 데이터 스트림을 지속해서 실시간 처리하는 거소가 달리, 온디맨드 또는 예약에 따라 실행
