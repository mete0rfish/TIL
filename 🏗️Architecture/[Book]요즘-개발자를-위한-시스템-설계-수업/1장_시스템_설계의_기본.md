# 1.1 시스템 설계의 정의
## 1.1.1 소프트웨어 시스템
특정 작업이나 일련의 작업을 수행하려고 함께 동작하는 컴포넌트나 모듈, 프로그램의 집합을 의미.
데이터 관리, 트랜잭션 처리, 최종 사용자를 위한 서비스 제공 등 원하는 기능을 수행하는 소프트웨어 애플리케이션

## 1.1.2 분산 소프트웨어 시스템
여러 독립적인 컴포넌트나 프로세스, 노드로 구성할 수 있는데, 이들은 서로 통신을 주고받으며 하나의 공동 목표를 달성하려고 동작.
- 통신 프로토콜
  - 원격 프로시저 호출 (RPC)
  - 메시지 전달
  - 발행/구독 모델
- 예시
  - 클라우드 컴퓨팅 플랫폼
  - P2P 네트워크
  - 분산 데이터베이스
  - CDN

## 1.1.3 시스템 설계의 이해
- 시스템 설계 과정
    1. 요구 사ㅏㅇ 분석
    2. 상위 수준의 아키텍처 설계: 시스템 전반 구조. 컴포넌트, 모듈, 인터페이스 등
    3. 하위 수준의 아키텍처 설계: 시스템 내부 구조. 각 컴포넌트 핵심 비즈니스 로직, 컴포넌트 간 상호 작용
    4. 사용자 인터페이스 설계
    5. API 설계
    6. 데이터베이스 설계

# 1.2 시스템 설계의 다양한 유형
## 1.2.1 상위 수준의 시스템 설계
### 시스템 아키텍처
- 시스템 아키텍처
  - 모놀리식
  - 클라이언트-서버
  - 마이크로서비스
  - 이벤트 주도
- 고려사항
  - 확장성
  - 유지보수성
  - 신뢰성
  - 지연 시간

### 데이터 흐름
- 데이터 수집: 데이터가 어디에서 오는지, 이를 시스템으로 어떻게 가져올지
- 데이터 저장: 데이터가 얼마나 자주 접근하는지, 검색 성능이 중요한지, 데이터 일솬성이 중요한지를 고려
- 데이터 처리
- 데이터 검색: 응답 시간이 얼마나 빠른지, 캐시를 사용해 속도를 높일지, 부하를 어떻게 분산할지를 고려

### 확장성
- 수직 확장성
- 수평 확장성

### 장애 허용 시스템
복제, 중복성, 점진적 성능 저하, 모니터링, 자가 복구 등으로 장애를 허용하여 신뢰성 높고, 다운타임이 적은 시스템을 설계하는 것

## 1.2.2 하위 수준의 상세 설계
### 알고리즘
- 시간 복잡도
- 공간 복잡도
- 트레이드 오프

### 데이터 구조
- 데이터 접근 방식
- 쿼리 성능
- 메모리 사용량

### API
- 일관성
- 유연성
- 보안
- 성능

### 코드 최적화
- 리펙터링
- 루프 언롤링
- 메모이제이션
- 병렬 처리

# 1.3 업계에서 시트메 설계가 갖는 중요성
- 요구 사항에 대한 명확한 이해: 요구 사항에 따라 시스템 핵심 기능, 성능, 보안 지침 등을 고려하여 문제를 해결하는 아키텍처 구축
- 협업의 시너지 효과: 모든 팀원이 시스템 설계에 참여하지 않지만, 이를 공유하여 소통과 조율이 원활히 함

# 1.4 시스템 설계으 중요성을 엿볼 수 있는 실제 사례
- 금융: 거래 내역, 고객 계좌 등 안정성, 신뢰성, 효율성
- 전자상거래: 온라인 거래, 재고 관리, 배송, 고객 정보 등 사용자 친화적이고 안전, 확장 가능성
- 의료: 안정성 신뢰성, 규제 준수
- 제조업: 재고 관리 등
