# 데이터 타입 (Collection)
1. String
2. Set
3. Sorted Set
4. Hash
5. List

## String
- 일반적인 문자열
- 최대 512 MB
- Binary Data, 비정형 데이터 저장 가능
- 단순 증감 연산에 좋음

## List
- 배열 구조
- 순서대로 저장
- 링크드 리스트 구조
- 추가, 삭제, 조회: O(1)
- 중간 index 조회: O(N)
- 메시지 큐로 적절

### 문법
```redis
# 왼쪽 오른쪽 푸쉬 팝
LPUSH <key> <value>
RPUSH <key> <value>
LPOP <key>
RPOP <key>

LLEN <key>
LREM <key> <count> <value>
```

## Hash
- field-value 구성
- key 하위에 subkey를 이용해 추가적인 해시 테이블 제공
- 제한없이 field 넣을 수 있음

## Set
- 중복 제거
- 정렬되지 않은 집합
- 마지막 넣은게 저장됨
- Set 간 교집합, 합집합, 차집합을 빠르게 계산
- 단, 모든 데이터를 갖고오는 명령이 있어 주의

## Sorted Set
- set에 score라는 필드 추가
- socre 순으로 정렬되어 저장
- 오름차순으로 내부 정렬
- value 중복 X, Score 중복 O
- 만약 score 값이 같으면, 사전 순으로 정렬되어 저장

## HyperLogLogs
- 대용량 데이터를 dump할때 사용
- 중복되지 않는 대용량 데이터를 count할 때 사용
- set과 비슷하지만 용량이 매우 작음
- 저장된 데이터 다시 확인 불가능

## Streams
- log를 저장하기 좋음
- append-only이며 중간에 데이터 안바뀜
- 읽어 올때 id 값 기반으로 시간 범위 검색
- tail -f로 신규 추가 데이터 수신
