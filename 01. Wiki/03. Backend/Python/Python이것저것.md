## 목차


## 1. __init__.py란
과거 파이썬(3.3 이전)에서는 폴더 안에 이 파일이 있어야만 해당 폴더를 **패키지**로 인식한다. 최근 버전에서는 이 파일이 없어도 '네임스페이스 패키지'로 인식된다.

### 그럼에도 사용 이유
- 패키지 초기화: 패키지가 임포트될 때 가장 먼저 실행. 패키지 수준에서 필요한 변수를 설정하거나 설정을 로드할 때 사용.

- 임포트 경로 간소화: 외부 사용자가 `from package.module import function` 대신 `from package import function`으로 바로 가져올 수 있도록 경로를 연결

- 패키지 심볼 관리: `__all__` 변수를 정의하여 `from package import *`를 했을 때 노출될 모듈을 제한할 수 있음


## 2. __pycache__ 폴더
파이썬이 소스 코드(.py)를 실행하기 직전 단계인 **바이트코드**로 변환하여 저장해두는 공간.

### 왜 생기는가?
파이썬 인터프리터는 코드를 `.pyc 파일`(컴파일된 파이썬 파일)로 변환하여 저장

### 장점
다음에 같은 프로그램을 실행할 때, 소스 코드가 수정되지 않았다면 이전에 만들어둔 `.pyc` 파일을 바로 사용한다. 덕분에 프로그램 로딩 속도가 훨씬 빨라진다.

### 삭제해도 되는가?
삭제해도 아무런 문제가 없다. 파이썬이 필요할 때 다시 자동으로 생성하기 때문이다. 다만, Git과 같은 버전 관리 시스템을 쓸 때는 보통 .gitignore에 등록해서 업로드되지 않도록 관리한다.
